[English]
*HOW TO RUN IT FOR THE FIRST TIME*

1) Change database connection string in Web.config (using your server and catalog)

2) Enable migrations in Persistance
enable-migration

3) Delete existing migrations and run again
Add-migration Initialize

4) Update database
update-database

5) Select default execution project
WebApp (you may change its name)

*CUSTOMIZE* 

1) Create your classes in the Domain folder in the Models project 
  Optionally you can create Grid in the Custom folder to view the data of your classes more friendly 
  
2) Add the Domain classes you want to save in the Database by adding the DbSet<T> field 
  in the ApplicationDbContext class inside the Persistence project 
  
3) Create the services for accessing the database and the controllers to display them 
  in views generated by you or create them automatically from the controllers

4) After changing the classes of the Domain folder you must perform a new migration


[Español]
*COMO EJECUTARLO POR PRIMERA VEZ*

1) Cambiar cadena de conexion de base de datos en Web.config (el server y el catalog)

2) Habilitar migraciones en Persistance
enable-migration

3) Borrar las migraciones existentes y ejecutar de nuevo
Add-migration Initalize

4) Actualizar base de datos
update-database

5) Seleccionar proyecto de ejecución por defecto 
WebApp (puedes cambairle el nombre al proyecto)

*PERSONALIZAR*
1) Cree sus clases en la carpeta Domain en el poryecto Models
  Opcionalmente puede crear Grid en la carpeta Custom para visualizar los datos de sus clases mas amigablemente
  
2) Agregar las clases de Domain que desee guardar en la Base de Datos agregando el campo DbSet<T>
  en la clase ApplicationDbContext dentro del proyecto Persistence
  
3) Cree los servicios para acceso a la base de datos y los controladores para mostrarlos
  en vistas generadas por usted o crearlas automaticamente desde los controladores

4) Luego de cambiar las clases de la carpeta Domain debe realizar una nueva migración
